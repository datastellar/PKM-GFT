# B4 — Mapping SPBE Challenges in Indonesia (Putra et al., 2023)

## 📖 Kutipan Lengkap (Harvard Style)
> Putra, A. S., Santoso, H. B., & Hasibuan, Z. A. (2023). Mapping SPBE Challenges in Indonesia: A Systematic Review by Architecture Domain. *Jurnal Sistem Informasi*, 19(1), 1–18. https://www.researchgate.net/publication/392916007_Mapping_SPBE_Challenges_in_Indonesia_Systematic_Review_by_Architecture_Domain

---

## 📌 Ringkasan
Paper ini melakukan **systematic literature review** terhadap tantangan implementasi SPBE di Indonesia, dipetakan berdasarkan domain arsitektur (infrastruktur, aplikasi, layanan, data, proses bisnis, keamanan, dll.). Temuan utama:

- **Fragmentasi sistem** adalah tantangan terbesar: setiap kementerian membangun sistemnya sendiri
- **Kurangnya standar interoperabilitas** yang mengikat antar instansi
- **Silo data** yang mengakibatkan duplikasi dan inkonsistensi
- **Resistensi SDM** terhadap digitalisasi dan perubahan proses bisnis
- **Anggaran tidak efisien** karena tidak ada koordinasi pengembangan TI

### Untuk Bab 1 (Pendahuluan) — Identifikasi Masalah ⭐ SANGAT PENTING
- Paper ini adalah **referensi akademik terkuat** untuk menggambarkan kondisi terkini: "Fragmentasi sistem dan kurangnya interoperabilitas adalah tantangan utama SPBE Indonesia" (Putra et al., 2023).
- Bisa dikutip sebagai bukti bahwa masalah ini sudah terdokumentasi secara ilmiah, bukan sekadar asumsi.

### Untuk Bab 2 (Gagasan) — Kerangka Masalah
- Pemetaan tantangan berdasarkan domain arsitektur → bisa digunakan untuk menunjukkan di mana DAMA-DMBOK memberikan solusi (misal: domain Data → DAMA knowledge areas).
- "Tantangan pada domain data dalam SPBE, seperti inkonsistensi dan duplikasi data, secara langsung dapat diatasi dengan pendekatan DAMA-DMBOK melalui Master Data Management dan Data Integration" (adaptasi dari Putra et al., 2023).

### Mapping Masalah ↔ Solusi DAMA:
| Tantangan (Putra et al., 2023) | Solusi DAMA-DMBOK |
|-------------------------------|-------------------|
| Fragmentasi sistem | Data Integration & Interoperability |
| Inkonsistensi data | Reference & Master Data |
| Tidak ada standar metadata | Metadata Management |
| Keamanan data lemah | Data Security |
| Duplikasi pengembangan | Data Architecture |
